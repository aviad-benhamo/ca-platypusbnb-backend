# Security Policy

## Supported Versions

The following versions of this project are currently supported with security updates:

| Version | Supported |
| ------- | --------- |
| < 1.0.x | Yes       |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please do not report it publicly via a GitHub issue or pull request. Instead, report it directly to the project owner.

Please send your security reports via email to: **aviadbenhamo@gmail.com** (or the email associated with the project owner on GitHub).

In your report, please include:
- A description of the vulnerability and its potential impact.
- Step-by-step instructions (or a proof-of-concept script/exploit) to reproduce the vulnerability.
- Any suggested remediation or fix.

### Expected Response Times

We take security vulnerabilities seriously and will make every effort to:
- Acknowledge receipt of your report within 48 hours.
- Provide a status update and estimated resolution timeframe within 7 days.
- Release a security patch or workaround as soon as practical, depending on the severity of the issue.

### Responsible Disclosure

We ask that you follow responsible disclosure guidelines, giving us a reasonable amount of time to resolve the issue before making any details public.
